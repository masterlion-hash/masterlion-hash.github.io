🔐 Flagship Security Project

SOC Agent — Autonomous Security Investigation System

An agentic AI system that autonomously investigates security alerts from an Elasticsearch-based SIEM. The agent polls anomaly alerts generated by ML jobs, then runs a multi-phase investigation pipeline: scoring correlations, querying logs for evidence, analyzing findings with an LLM (Ollama), and producing a final verdict with severity classification. Features a Think→Act→Observe reasoning loop, safety-gated response tiers (from logging to containment), and a WhatsApp integration (OpenClaw) that delivers real-time alerts to SOC analysts for approve/reject decisions.

PythonElasticsearchLangChain OllamaAgentic AISIEM WhatsApp APICybersecurity

⚙ Investigation Pipeline

1
Poll Anomaly Alerts Continuously queries Elasticsearch ML job results for triggered anomaly scores above threshold
2
Score & Correlate Cross-references alerts with historical data to compute composite risk scores and correlation patterns
3
Query Log Evidence Fetches supporting log entries, network events, and host telemetry from the SIEM for context
4
LLM Analysis (Ollama) Think→Act→Observe loop: the agent reasons over evidence, decides on follow-up queries, then synthesizes findings
5
Verdict & Analyst Alert Produces severity classification and pushes a structured alert to analysts via WhatsApp (OpenClaw) for approve/reject

⚡ Key Capabilities

  • 🧠 Autonomous reasoning loop — Think→Act→Observe cycle enables multi-step evidence gathering without human prompting at each step
  • 🔒 Safety-gated response tiers — graduated actions from passive logging and enrichment, up to active containment, each requiring analyst approval before escalation
  • 📱 WhatsApp analyst bridge — real-time alert delivery via OpenClaw integration; analysts approve or reject containment actions directly from their phone
  • 📊 Severity classification — LLM-generated verdict includes confidence score, MITRE ATT&CK mapping suggestion, and recommended next steps
  • 🔎 Full audit trail — every reasoning step, query, and decision is logged for post-incident review and model improvement
